# Claude Code 2.1.70 Release Notes

**Release Date:** March 6, 2026
**Version:** 2.1.70 (Claude Code)

## Overview

Claude Code 2.1.70 is a stable release featuring comprehensive command-line interface capabilities for developers. This version provides interactive and non-interactive modes, session management, advanced permissions handling, and deep integration with development workflows.

## Key Features

### Core Capabilities
- **Interactive Sessions**: Conversational interface with context persistence
- **Print Mode**: Non-interactive output for scripting and automation
- **Session Management**: Save, resume, and fork conversations
- **Model Selection**: Choose between Haiku (fast), Sonnet (balanced), and Opus (powerful)
- **Custom Agents**: Define specialized agents for specific roles
- **Permission Modes**: Flexible security from fully interactive to automatic
- **Git Integration**: Native worktree support for isolated development

### Advanced Features
- **MCP Server Support**: Extend capabilities with Model Context Protocol
- **Structured Output**: JSON validation and streaming
- **IDE Integration**: Direct connection to supported IDEs
- **Debug Logging**: Comprehensive debugging capabilities
- **Budget Control**: Spending limits and cost management
- **Tool Customization**: Whitelist or blacklist available tools

## Command Reference

All commands follow the pattern:
```
claude [options] [command] [prompt]
```

### Main Commands
- **Interactive**: `claude` or `claude "prompt"`
- **Print Mode**: `claude -p "prompt"`
- **Continue**: `claude -c` (resume last session)
- **Resume**: `claude -r [session-id]`
- **Worktree**: `claude --worktree name`

### Available System Commands
- `agents` - List configured agents
- `auth` - Manage authentication
- `doctor` - Check auto-updater health
- `install` - Install Claude Code native build
- `mcp` - Configure MCP servers
- `plugin` - Manage plugins
- `setup-token` - Create long-lived auth token
- `update|upgrade` - Check and install updates

### Major Options

**Model Selection**
```
--model haiku      # Fast, suitable for quick tasks
--model sonnet     # Balanced (default)
--model opus       # Most capable, slower
```

**Permission Modes**
```
--permission-mode default          # Ask for each action
--permission-mode plan             # Suggest changes (you approve)
--permission-mode acceptEdits      # Auto-apply edits
--permission-mode dontAsk          # Full automation
--permission-mode bypassPermissions # Bypass all (sandbox only)
```

**Output Control**
```
--output-format text       # Plain text (default)
--output-format json       # JSON output
--output-format stream-json # Real-time streaming
```

**Security & Access**
```
--add-dir /path              # Allow directory access
--allowedTools "Bash,Edit"   # Whitelist tools
--disallowedTools "Bash(rm)" # Blacklist tools
--dangerously-skip-permissions # Bypass checks (sandbox only)
```

**Development**
```
--worktree [name]          # Create isolated git worktree
--tmux                     # Use tmux for worktree (with --worktree)
--fork-session             # Create new session ID when resuming
--debug [filter]           # Enable debug logging
--debug-file <path>        # Write debug logs to file
```

**Advanced**
```
--agents '{"role": {...}}'     # Define custom agents
--agent name                   # Use specific agent
--system-prompt "text"         # Custom system instructions
--append-system-prompt "text"  # Add to default system prompt
--json-schema '{...}'          # Validate JSON output
--max-budget-usd <amount>      # Limit API spending
--session-id <uuid>            # Use specific session ID
--mcp-config <files>           # Load MCP servers
--file <specs>                 # Download file resources
```

## Usage Patterns

### Interactive Development
```bash
# Start a conversation
claude

# Or with initial prompt
claude "Help me debug this issue"

# Continue later
claude -c

# Resume specific session
claude -r session-id
```

### Scripting & Automation
```bash
# Quick answer
claude -p "What is REST API?"

# Generate code
claude -p "Write a Python function to reverse a string"

# Process with pipe
cat data.json | claude -p "Convert this JSON to YAML"

# Save to file
claude -p "Generate boilerplate" > output.txt
```

### Code Analysis
```bash
# Analyze file
claude -p "Review this code:" < myfile.js

# Review directory
claude -p "Architecture review" --add-dir ./src

# Security audit
claude -p "Security analysis" < app.py
```

### Development Workflows
```bash
# Create isolated environment
claude --worktree feature-xyz

# Safe mode refactoring
claude --permission-mode plan "Refactor this code"

# Expert consultation
claude --system-prompt "You are a security expert" "Review for vulnerabilities"
```

## Best Practices

### Prompting
✅ **DO:**
- Be specific about requirements
- Provide context and examples
- State desired output format
- Use interactive mode for refinement

❌ **DON'T:**
- Use vague language
- Skip context
- Assume understanding
- Try everything in print mode

### Permission Management
✅ **DO:**
- Use `--permission-mode plan` for code changes
- Review suggested edits
- Use tool restrictions when appropriate

❌ **DON'T:**
- Use `bypassPermissions` outside sandboxes
- Skip permission reviews
- Trust all outputs automatically

### Session Management
✅ **DO:**
- Use meaningful session IDs
- Resume for continuous work
- Fork to explore alternatives
- Clean up completed sessions

❌ **DON'T:**
- Lose context by always starting fresh
- Keep session clutter
- Ignore session history

### Scripting
✅ **DO:**
- Use `-p` mode for automation
- Add error handling
- Validate output
- Use appropriate models

❌ **DON'T:**
- Use interactive mode in scripts
- Assume commands always succeed
- Overuse expensive models
- Ignore edge cases

## Security Considerations

### Data Privacy
- Sessions are stored locally by default
- No data sent beyond Claude's servers
- Sensitive data handling depends on your prompt

### Tool Permissions
- By default, all safe tools are available
- Use `--allowedTools` to restrict
- Use `--disallowedTools` to blacklist
- Review tool use in permission mode

### API Key Management
- Use `setup-token` for long-lived tokens
- Keep tokens in secure config
- Use `--max-budget-usd` to limit exposure
- Monitor API usage regularly

4. **Model Availability**: Model availability depends on account
5. **Network Dependency**: Requires internet connection

## Troubleshooting

### Common Issues

**Issue:** Claude isn't responding
- **Solution:** Check internet connection and try again

**Issue:** Permission denied errors
- **Solution:** Check `--permission-mode` or `--allowedTools` settings

**Issue:** Slow responses
- **Solution:** Try `--fallback-model` or use Haiku

**Issue:** Out of budget
- **Solution:** Set `--max-budget-usd` or use cheaper models

**Issue:** Session not found
- **Solution:** List available sessions with `claude -r`
